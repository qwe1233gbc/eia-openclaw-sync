## 一、v0.5 Error Analysis

### 1.1 错误分类体系

对 v0.5 24 组回答（8题×3组）逐一审查，将问题分为四类：

| 错误类型 | 定义 | 出现次数 | 影响题目 |
|---------|------|---------|---------|
| **检索误配** (retrieval_mismatch) | 经验卡被关键词匹配但却不适用于当前审核任务 | 1 | Q03 |
| **覆盖面空白** (coverage_gap) | 经验库无对应卡片，C组=B组，增量=0 | 2 | Q04, Q08 |
| **低区分度** (low_discrimination) | A/B/C三组均为满分，无法测量增量 | 1 | Q08 |
| **审核意见缺损** (comment_deficiency) | A组缺少标准级核查要求，comment不完整 | 4 | Q03, Q04, Q05, Q06 |

## 四、经验卡 Relevance 判别机制设计

### 4.1 问题总结

v0.5 唯一的检索错误：EXP_01（胶水VOCs含量）在 Q03（涂胶量核算）中被低关联命中。根因是纯关键词匹配（"聚氨酯胶水"）无法区分"VOCs含量审核"和"涂胶量计算"两个不同任务。

### 4.2 三层 Relevance 判别架构

```text
Relevance Score = α × keyword_match + β × task_type_match + γ × module_match

α = 0.2 (关键词相似度权重)
β = 0.5 (任务类型匹配权重) ← 核心权重
γ = 0.3 (审核模块匹配权重)
```

#### 第一层：keyword_match（0-1）

| 匹配方式 | 权重 | 示例 |
|---------|------|------|
| 精确实体匹配（"聚氨酯胶水"→"聚氨酯胶水"） | 0.4 | EXP_01→Q03 |
| 概念匹配（"胶粘剂"→"胶水"） | 0.3 | EXP_01→Q01 |
| 场景匹配（"涂布工序"→"涂胶工序"） | 0.3 | EXP_04→Q07 |

#### 第二层：task_type_match（0-1）← **新增字段**

每张经验卡新增 `applicable_task_types` 字段：

```json
{
  "experience_card_id": "EXP_01",
  "applicable_task_types": ["guide_explicit", "case_experience"],
  "not_applicable_task_types": ["standard_calculation"]
}
```

| 经验卡 | applicable_task_types | 说明 |
|--------|----------------------|------|
| EXP_01 | guide_explicit, case_experience | VOCs含量限值判断+类案类比 |
| EXP_02 | guide_explicit, standard_calculation | 更换周期审查+公式验证 |
| EXP_03 | case_experience, cross_check | 收集效率类比+交叉核验 |
| EXP_04 | case_experience, cross_check | 工艺遗漏类比+交叉核验 |
| EXP_05 | case_experience, cross_check | 危废遗漏类比+交叉核验 |
| EXP_06 | case_experience, guide_explicit | 总量替代类比+明文审查 |

匹配规则：
- 当前题目的 `task_type` ∈ 经验卡的 `applicable_task_types` → task_type_match = 1.0
- 当前题目的 `task_type` ∈ 经验卡的 `not_applicable_task_types` → task_type_match = 0（直接丢弃）
- 当前题目的 `task_type` 不在以上两者 → task_type_match = 0.5（中性）

**Q03 案例回放**：
- Q03 task_type = "standard_calculation"
- EXP_01 applicable_task_types = ["guide_explicit", "case_experience"]
- Q03 task_type ∉ EXP_01 applicable_task_types
- → task_type_match = 0.5（中性，不是完全不匹配）
- → keyword_match = 0.7（"聚氨酯胶水"关键词命中）
- → module_match = 0.3（Q03模块=涂胶量核算，EXP_01模块=胶粘剂VOCs含量）
- → Relevance = 0.2×0.7 + 0.5×0.5 + 0.3×0.3 = 0.14+0.25+0.09 = 0.48

如果 EXP_01 设置 `not_applicable_task_types: ["standard_calculation"]`：
- → task_type_match = 0 → Relevance = 0.14+0+0.09 = 0.23 → 丢弃

#### 第三层：module_match（0-1）

新增 `applicable_modules` 字段：

```json
{
  "experience_card_id": "EXP_01",
  "applicable_modules": ["原辅材料-VOCs", "胶粘剂"],
  "applicable_task_types": ["guide_explicit", "case_experience"]
}
```

module_match 计算：
- question_module 与 experience_card.applicable_modules 的 Jaccard 相似度

### 4.3 三级 Relevance 判别

| 级别 | Relevance Score | 行为 |
|------|----------------|------|
| **HIGH** | ≥ 0.60 | 可支撑判断，标注为类案依据 |
| **MEDIUM** | 0.35–0.59 | 仅作为辅助提示，标注"需人工判断适用性" |
| **LOW** | < 0.35 | 丢弃，不在审核意见中引用 |

### 4.4 实施要求

1. 为全部 8 条经验卡补充 `applicable_task_types` 和 `applicable_modules` 字段
2. 在 Skill 的 Step 4（经验库检索）中增加 Relevance 计算步骤
3. 在输出 JSONL 中新增 `relevance_score` 和 `relevance_level` 字段
4. 在自反思中增加 `relevance_check`：检查引用的经验卡 relevance 是否 ≥ MEDIUM
