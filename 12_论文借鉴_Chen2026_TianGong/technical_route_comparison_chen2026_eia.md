# Chen 2026 与本课题技术路线对照整理

> 用途：整理 Chen 2026 的技术路线，并基于本仓库当前已有数据，形成“塑胶行业环评审核技能库”课题的对应技术路线。
>
> 边界说明：本课题当前不做模型微调。本文借鉴 Chen 2026 的核心是“权威材料结构化、混合检索、工作流部署、专家验证和分层评价”，不是照搬其 fine-tuning 路线。

## 1. Chen 2026 的技术路线

Chen 2026 的研究目标可以概括为：围绕环境复杂性问题，构建结构化环境问答数据集，并比较不同大模型部署策略在环境知识任务中的表现。

其技术路线可拆为 7 个环节。

### 1.1 权威环境语料汇集

Chen 2026 首先汇集环境领域权威教材和补充材料，覆盖多个环境子领域和较长时间跨度。其目的不是单纯堆积文本，而是形成可支撑环境知识问答、推理和评价的权威语料基础。

技术含义：

- 明确语料来源和覆盖范围；
- 区分权威材料与普通网页材料；
- 为后续检索、数据生成和专家验证提供事实基础。

### 1.2 非结构化文档解析与切块

论文及开源项目中体现了非结构化文档处理思路：将教材或文档解析为可检索的文本片段，保留来源、章节、页码、chunk 等 metadata。

技术含义：

- OCR / 文档解析；
- 文本切块；
- metadata 记录；
- 后续可回溯到原始来源。

### 1.3 混合检索索引构建

Chen 2026 相关开源项目中包含向量数据库和全文检索思路。向量检索用于语义相似召回，全文检索用于标准号、术语、关键词等精确查询。

技术含义：

- 向量索引：解决语义相似问题；
- 全文索引：解决术语、编号、标准名称等精确召回问题；
- 混合召回：降低单一路径检索失败风险。

### 1.4 种子问题驱动的数据生成

在数据构建阶段，Chen 2026 使用种子问题检索相关教材片段，再由大模型生成结构化问答。每条生成记录通常包括问题、答案、推理痕迹和检索片段。

技术含义：

- 用问题驱动材料召回；
- 用 schema 约束生成结果；
- 生成内容必须绑定原始证据；
- 推理痕迹用于复核透明度，但不能替代证据。

### 1.5 专家参与的数据验证

Chen 2026 的重要特点是生成数据并非直接使用，而是经过 expert-in-the-loop 验证。SI 中说明，自动流程生成 19,739 条记录，经专家验证后保留 19,532 条，剔除 207 条。

专家主要检查：

- factual alignment：答案中的数值、概念、技术描述是否能由来源片段支撑；
- contextual relevance：问题是否与检索片段实质相关。

剔除类型包括：

- 空泛来源或无实质信息；
- 问题与来源不匹配；
- 无依据数值、概念或技术判断。

### 1.6 多种部署策略比较

Chen 2026 对比了不同模型或部署方式，包括基础模型、微调模型和 agentic workflow。对本课题最有启发的不是微调结果本身，而是它把任务分为不同处理方式：

| 任务类型 | Chen 2026 中的处理启发 |
| --- | --- |
| 稳定、标准化知识 | 可沉淀为结构化数据或规则 |
| 需要跨材料整合的问题 | 需要检索增强 |
| 需要计算、推理、工具调用的问题 | 适合工作流或 agentic workflow |
| 需要专业判断的问题 | 必须保留专家验证 |

### 1.7 测试集、评分和统计验证

Chen 2026 SI 中的测试集和评价框架包括：

- 测试题四维分类：认知层级、环境领域、推理类型、功能能力；
- 评分提示词：10-100 分分档；
- 错误类型分析：幻觉、事实漂移、上下文误读、推理链错误、数值传播错误；
- 统计验证：Shapiro-Wilk、Bootstrap 置信区间、paired t-test、Wilcoxon、FDR 校正、Cohen's d；
- 多评审者一致性：Fleiss' kappa，用于提示评分口径需要校准。

因此，Chen 2026 的完整技术路线可概括为：

```text
权威环境语料
  -> 文档解析与切块
  -> 向量检索 + 全文检索
  -> 种子问题驱动召回
  -> 结构化问答生成
  -> 专家验证与剔除
  -> 不同模型/工作流部署策略比较
  -> 分类测试集、评分量规、统计验证和错误分析
```

普通纯注塑项目若不涉及上述典型工艺和 VOCs 治理场景，建议只作为参照样本，不作为核心实验对象。

## 4. 本项目建议技术路线

本项目不进行微调，建议采用“标准依据库 + 审核技能库 + 混合检索 + 样本链证据 + 人工复核 + 知识条件对照实验”的技术路线。

完整路线如下：

```text
研究边界收敛
  -> 数据来源整理
  -> 标准依据库构建
  -> 审核技能库构建
  -> 样本链证据整理
  -> 混合检索索引构建
  -> 候选审核任务筛选与生成
  -> 人工复核与金标答案校准
  -> A/B/C/D 知识条件对照实验
  -> 错误案例分析与技能库迭代
```

### 4.1 第一步：研究边界收敛

目标：

- 明确本课题不是通用环评审核；
- 明确研究对象为佛山塑胶指南适用范围内的典型项目；
- 排除或降级纯注塑、电子电器、普通五金等不适合作为核心验证对象的材料。

当前已有基础：

- `00_先看这里_README/README.md` 已明确研究方向；
- `01_塑胶指南原文与适用范围/`、`02_适用行业与样本筛选/` 已提供边界材料；
- 进度报告已说明研究对象已收敛。

仍需补充：

- 对 C2921-C2929 及相邻行业边界进行人工复核；
- 建立“核心样本 / 参照样本 / 排除样本”的稳定判定表。

### 4.2 第二步：数据来源整理

目标：

- 把标准、报告、修改意见、批复、技能规则分成不同证据类型；
- 每个片段保留来源路径、项目 ID、材料类型、是否人工复核等 metadata。

当前已有基础：

- 标准库和 Dify 知识库已经初步整理；
- 样本链已按项目组织；
- QA 文件中已有 evidence、source_basis、manual_check 等字段。

仍需补充：

- QA 文件中部分人工审核列存在编码显示问题，需要统一字段名；
- 样本链 inventory 目前规模较小，不能覆盖全部 33 个项目目录；
- 报告片段尚未统一记录页码、章节、材料类型和复核状态。

### 4.3 第三步：标准依据库构建

目标：

- 将指南、标准、政策、产污系数、三线一单等材料转为可检索的标准卡；
- 每条标准卡明确适用范围、触发关键词、证据字段、判断逻辑、人工复核边界。

当前已有基础：

- `standard_cards.jsonl` 已有 37 条结构化标准卡；
- `plastic_guide_standard_library_v3_deep.jsonl` 有 71 条旧版条目；
- `cards_by_skill/` 已按 15 个 skill 拆分标准依据；
- `quality/QUALITY_CHECK.md` 已记录 GB3096、环保投资、活性炭参数等需校准点。

仍需补充：

- 合并 v3 与标准卡体系，避免重复和旧版条目混用；
- 优先校准 GB 31572、GB 37822、DB44/2367、产污系数、三线一单；
- 对每条标准卡补充版本有效性、强制性边界和人工复核状态。

### 4.4 第四步：审核技能库构建

目标：

- 把 Dify 工作流中的审核逻辑拆成可复用 skill；
- 每个 skill 明确输入材料、证据抽取、判断步骤、输出字段和人工复核条件。

当前已有基础：

- 已形成 15 个独立审核 skill 和 1 个汇总 skill；
- 已有 `WORKFLOW_MAPPING.md` 说明 Dify 工作流到 skill 的迁移；
- 已有统一 JSON 输出 schema 和 output example。

仍需补充：

- 用真实报告样本验证每个 skill 的输入字段是否足够；
- 检查 skill 输出是否能稳定形成“问题、证据、依据、结论、修改建议”；
- 将类案经验绑定到真实修改意见或批注后，再作为 skill 的辅助知识。

### 4.5 第五步：样本链证据整理

目标：

- 建立从受理公告报告到修改意见、终稿、批复的证据链；
- 用真实修改意见校准“模型应发现的问题”和“修改建议是否可执行”。

当前已有基础：

- `05_样本链（按项目组织）/` 已有 33 个项目目录；
- 已包含受理公告、终稿、修改意见、候选样本、排除样本等材料；
- `_元数据/` 下已有样本链说明和若干清单。

仍需补充：

- 完整链条数量不足；
- 典型涉胶水、复合、熟化、VOCs 治理项目仍需补强；
- 需要将每个真实修改意见拆成“问题类型、报告证据、标准依据、最终修正情况”。

### 4.6 第六步：混合检索索引构建

目标：

- 类比 Chen 的向量库 + 全文库，将本项目材料做成可检索语料；
- 支持按标准号、污染物、工艺、项目名、审核任务召回证据。

当前已有基础：

- `scripts/build_hybrid_index.py`；
- `scripts/hybrid_retrieve.py`；
- `configs/eia_plastic_hybrid_index.yaml`；
- `schemas/corpus_chunk.schema.json`。

仍需补充：

- 用当前标准库、技能库、QA、实验设计跑通索引构建；
- 将样本链材料纳入配置；
- 抽样检查检索结果是否能召回正确依据。

### 4.7 第七步：候选审核任务筛选与生成

目标：

- 从 942 条种子 QA 中筛选可评价任务；
- 每题绑定报告片段、标准依据、修改意见或批复来源；
- 标注认知层级、审核领域、推理类型、功能能力。

当前已有基础：

- 942 条种子 QA 覆盖 151 个项目；
- 已有 `generated_qa.schema.json` 和候选审核任务生成 prompt；
- 已有 Chen SI 四维分类迁移框架。

仍需补充：

- 先筛选 30-50 条证据链完整、边界清楚的题目；
- 修复或清理 QA 中的乱码字段；
- 建立 rejected log，记录证据不足、问题-来源不匹配、无依据数值等失败原因。

### 4.8 第八步：人工复核与金标答案校准

目标：

- 将候选任务变成正式评价题；
- 形成可比较的人工参考答案和评分依据。

当前已有基础：

- QA 中已有 manual_check 相关字段；
- 样本链中已有部分修改意见；
- `evaluation_record.schema.json` 已能记录评分和错误标签。

仍需补充：

- 至少完成一批 30-50 题人工复核；
- 对每题明确 retain / revise_needed / reject；
- 建立专家校准记录，尤其是评分口径不一致的题目。

### 4.9 第九步：A/B/C/D 知识条件对照实验

目标：

- 不比较“是否微调”，而比较不同知识输入条件对审核输出质量的影响。

建议实验组：

| 组别 | 输入条件 | 对应研究问题 |
| --- | --- | --- |
| A | 仅报告片段或审核问题 | 裸模型能否识别基本问题 |
| B | A + 标准依据库检索结果 | 标准依据是否提升事实扎根和标准适用性 |
| C | B + 审核技能库步骤 | 技能流程是否提升审核逻辑完整性 |
| D | C + 样本链/修改意见/类案经验 | 真实经验材料是否提升修改意见可执行性 |

评分维度建议：

- 判断正确性；
- 证据使用；
- 标准适用性；
- 审核逻辑；
- 数值或计算可靠性；
- 修改意见可执行性；
- 误报控制。

仍需补充：

- 当前只有实验设计，尚无正式评分结果；
- 需要人工评分和至少一次一致性校准；
- 样本量不足时只能报告阶段性观察，不能宣称已经证明有效。

| 统计验证未开展 | 有评分框架，无正式评分矩阵 | 不能报告显著性或效应量 | 完成人工评分后再做 Bootstrap/配对检验 |

## 7. 推荐写入论文的方法表述

可以采用如下表述：

> 本研究参考 Chen 2026 在环境领域大模型研究中提出的结构化知识构建、检索增强、工作流部署和专家验证思路，但不开展模型微调。研究以佛山塑胶行业环评审核为应用场景，将指南、标准规范、环评报告、修改意见和批复等材料组织为标准依据库、样本链和审核技能库，并通过混合检索与模块化审核流程生成候选审核意见。在评价阶段，本文设计 A/B/C/D 知识条件对照实验，从证据扎根、标准适用性、审核逻辑、数值可靠性和修改意见可执行性等维度评估不同知识输入对审核辅助质量的影响。

中期阶段应使用克制表述：

- 已初步形成技术路线；
- 已具备进一步实验基础；
- 仍需补充样本链；
- 仍需人工复核和专家校准；
- 当前不宜宣称已经证明系统有效。

## 8. 与 Chen 2026 的关键差异

| 维度 | Chen 2026 | 本课题 |
| --- | --- | --- |
| 数据对象 | 通用环境教材与环境知识问答 | 塑胶行业环评报告、标准、修改意见和审核任务 |
| 核心任务 | 环境知识问答、推理和模型能力评价 | 环评文件审核辅助和修改意见生成 |
| 训练路线 | 包含结构化数据集和微调模型比较 | 不做微调，采用 RAG + skill + 工作流 |
| 验证对象 | 问答答案是否由教材支撑 | 审核意见是否由报告、标准和修改意见支撑 |
| 评价设计 | 模型/工作流性能比较 | 知识条件对照实验 |
| 风险边界 | 数据幻觉、来源不匹配、评分一致性 | 标准误用、类案经验过度泛化、样本链不足、专家复核不足 |

因此，本课题的创新点不宜表述为算法原创，而应定位为：

> 面向塑胶行业环评审核场景的结构化知识组织、审核技能拆解和知识条件对照评价框架。
