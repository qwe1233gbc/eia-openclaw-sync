## 4. 论文结构建议

建议按以下框架组织论文：

```
第1章 绪论
  - 环评审核的两种知识来源：明文标准与类案经验
  - 研究问题：类案经验能否在标准库之外提供增量审核价值？

第2章 相关研究
  - 环评智能审核研究现状
  - 法律领域的判例推理与类案检索
  - 知识图谱与经验规则构建

第3章 方法
  3.1 标准依据库构建：从法规/标准/导则/管理文件中抽取条款
  3.2 类案经验库构建：从真实审核批注中抽象经验规则
  3.3 证据分级体系：A/B/C三级
  3.4 经验规则质量评估维度：完整性、标准支撑度、迁移性、可靠性

第4章 构建结果
  4.1 标准依据库：2544条条款（from reports）+ 12份标准解析文件
  4.2 类案经验库：176条规则（12A/7B/157C），6条核心C2929类案对
  4.3 规则质量分层分析：类案对 vs 通用模板的差异

第5章 实验验证
  5.1 C2929注塑行业benchmark构建（30-50题）
  5.2 三组对照实验（纯标准库/标准库+模板/标准库+经验库）
  5.3 实验结果：问题识别率、误报率、人机一致性（Kappa）
  5.4 案例深度分析：盛之强案例的三组对比

第6章 讨论
  6.1 类案经验库的增量价值与适用边界
  6.2 经验规则的可迁移性（C2929→C2921/C2922）
  6.3 当前局限与未来工作

第7章 结论
```

## 5. 不推荐做的事情（反规划）

以下事情**不建议**在当前阶段做：

| 不建议做的事 | 原因 |
|-------------|------|
| 把157条C级规则全部校准 | 模板化严重，ROI极低。不如集中精力做好6条强规则 |
| 扩展到所有行业 | C2929是目前唯一有真实批注和完整支持文件的行业 |
| 构建"全自动审核系统" | 论文研究的是"经验库是否有效"，不是"系统能否替代人" |
| 刷benchmark数量到100+ | 30-50条精心标注的benchmark比100+条质量参差的有价值 |
| 追求"系统能生成文字就是有效" | 必须通过对照实验验证增量价值，不能用系统有输出就声称有效 |
| 把C2929经验直接迁移到其他行业 | 论文应坦诚讨论迁移的局限性和条件 |
