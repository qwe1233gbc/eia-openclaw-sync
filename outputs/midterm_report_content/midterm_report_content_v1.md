### 第 3 页：研究背景

标题：

```text
环评审核正在从文本审查转向证据化审查
```

正文：

```text
建设项目环评文件审核需要同时核对项目事实、行业分类、标准适用、污染源强、治理设施和管理要求。

在实际审核中，问题往往不是单一条款判断，而是跨章节、跨标准、跨版本材料的综合核验。

因此，环评审核辅助工具不能只给出结论，还需要说明依据来源、证据片段和人工复核边界。
```

页面可视化建议：

```text
报告事实 → 标准依据 → 审核判断 → 修改意见 → 人工复核
```

口头表达建议：

> 环评审核并不是简单问答。一个审核意见通常需要回到报告原文、标准条款、行业边界和修改意见之间反复核对。因此，本研究强调证据可追溯，而不是让模型直接给出不可复核的结论。

口头表达建议：

### 第 5 页：技术背景

标题：

```text
大模型可辅助审核，但必须解决依据和复核问题
```

正文：

```text
大模型具备较强的文本理解和归纳能力，可用于报告内容抽取、标准检索、问题识别和修改意见生成。

但直接使用大模型进行环评审核存在三类风险：
- 依据幻觉：生成标准中不存在的限值、条款或适用条件；
- 经验泛化：将个案经验误当作普遍强制规则；
- 复核困难：无法说明结论来自哪一份报告、哪一条标准或哪一条修改意见。

因此，本研究采用“检索增强 + 审核技能库 + 人工复核”的辅助路线。
```

口头表达建议：

> 大模型可以提升文本处理效率，但环评审核是高准确性要求的技术场景，不能把模型输出直接当成审批结论。我的研究重点是构建可追溯、可复核的审核辅助流程。

### 第 7 页：研究空白

标题：

```text
现有研究尚未解决地方行业环评审核的证据链问题
```

正文：

```text
现有大模型环境应用研究多集中于通用环境问答、政策解读或文本生成。

对于地方行业环评审核，仍存在不足：
- 缺少面向具体行业指南的标准依据组织方法；
- 缺少报告、修改意见、终稿和批复之间的样本链；
- 缺少将审核经验拆解为可复用技能的结构；
- 缺少不同知识输入条件下的审核质量评价。

本研究拟补充的是场景化、证据化、可复核的审核辅助方法。
```

口头表达建议：

> 环评审核最难的部分不只是“回答问题”，而是判断答案是否能被报告原文、标准依据和真实修改意见支持。这个证据链问题，是本课题区别于通用环保问答的关键。

### 第 14 页：评价设计

标题：

```text

```

正文：

```text
实验组别：
A 组：仅报告片段或审核问题
B 组：A + 标准依据库
C 组：B + 审核技能库步骤
D 组：C + 样本链、修改意见或类案经验

评分维度：
- 判断正确性；
- 证据使用；
- 标准适用性；
- 审核逻辑；
- 数值或计算可靠性；
- 修改意见可执行性；
- 误报控制。

当前状态：
实验方案已形成，正式评分仍需等待评价题筛选和人工复核。
```

口头表达建议：

> 我不比较微调模型，而是比较不同知识条件是否会改善审核输出。例如，加入标准库是否减少标准适用错误，加入技能库是否提高审核逻辑完整性。
