# 基于 Chen 2026 SI 的环评审核评价验证框架

> 用途：把 Chen 2026 Supporting Information 中的训练数据验证、测试集分类、评分提示词、统计检验和错误案例分析，迁移为本课题“塑胶行业环评审核技能库”的后续评价方案。

## 1. 本课题评价对象

本框架不评价“自动审批能力”，只评价以下阶段性研究对象：

1. 标准依据库条目是否能支撑审核判断；
2. 审核技能库输出是否能给出可追溯、可复核的审核意见；
3. QA/审核任务样本是否具备作为对照实验材料的条件；
4. 不同知识条件输入下，模型或工作流输出质量是否存在可观察差异。

## 2. 样本分类框架

建议每道审核任务都标注四类标签。

### 2.1 认知层级

| 标签 | 含义 | 示例 |
| --- | --- | --- |
| L1_understanding | 标准、术语、边界的识别和解释 | 判断项目是否属于佛山塑胶指南适用范围 |
| L2_application | 将标准或规则应用到具体报告事实 | 根据原辅料和工艺判断是否需要 VOCs 源强核算 |
| L3_analysis | 多证据、多标准、多版本材料的综合判断 | 对比受理稿、终稿、修改意见，判断问题是否已闭合 |

### 2.2 审核领域

建议使用以下领域标签：

- `industry_boundary`
- `three_lines_one_permit`
- `construction_content`
- `environmental_quality`
- `emission_standard`
- `pollution_coefficient`
- `source_strength`
- `gas_collection`
- `treatment_facility`
- `activated_carbon`
- `solid_waste_hazardous_waste`
- `total_amount_control`
- `monitoring_and_management`
- `sample_chain_consistency`

### 2.3 推理类型

| 标签 | 含义 |
| --- | --- |
| factual_descriptive | 抽取事实、列出依据、解释概念 |
| mechanistic | 说明污染产生、收集、治理、转移或失效机理 |
| quantitative | 涉及限值、系数、风量、浓度、总量、危废量等计算 |
| comparative | 比较标准、方案、版本或工艺适用性 |
| cross_document_synthesis | 需要跨标准库、报告原文、修改意见、批复或类案经验综合判断 |

### 2.4 功能能力

| 标签 | 含义 |
| --- | --- |
| knowledge_grounding | 标准、术语和事实依据是否正确 |
| numerical_fidelity | 数值、单位、公式和中间过程是否可靠 |
| logical_reasoning | 判断链是否完整、是否跳步 |
| evidence_integration | 是否整合多来源证据并说明冲突 |
| review_comment_actionability | 修改意见是否具体、可执行、不过度泛化 |

## 3. 人工复核流程

借鉴 SI 的 expert-in-the-loop 思路，每条候选审核任务建议保留以下材料：

| 字段 | 说明 |
| --- | --- |
| seed_question | 原始审核问题 |
| retrieved_chunks | 检索到的标准、报告、修改意见、类案片段 |
| generated_answer | 模型或工作流输出 |
| reasoning_trace | 中间推理，仅用于复核，不作为最终证据 |
| evidence_alignment | 事实与证据是否一致 |
| contextual_relevance | 问题和证据是否匹配 |
| reviewer_decision | retain / revise_needed / reject |
| rejection_reason | 若剔除，记录失败类型 |

复核原则：

1. 答案中所有关键事实、数值、标准名称、工艺判断必须能回到证据；
2. 检索证据仅为目录、标题、索引、空泛描述时，不得生成具体审核结论；
3. 类案经验只能作为辅助判断，不能替代标准或项目原文；
4. 需要计算的问题必须保留公式、单位和关键中间值；
5. 对无法确认的内容，应输出“需人工复核”，而不是补写确定性结论。

## 4. 拒收规则

建议将以下情况直接标记为 `rejected` 或 `manual_rebuild_needed`：

| 拒收类型 | 判定标准 |
| --- | --- |
| `vacuous_source` | 证据片段没有实质内容，无法回答问题 |
| `question_source_mismatch` | 问题需要的数据或依据不在证据片段中 |
| `unsupported_numeric_claim` | 出现证据中没有的限值、效率、系数、浓度、风量、总量 |
| `unsupported_standard_claim` | 标准适用性判断没有条款或适用条件支撑 |
| `contextual_misinterpretation` | 误读项目行业、工艺、原辅料或治理设施 |
| `case_overgeneralization` | 将类案经验当作普遍强制规则 |
| `calculation_chain_error` | 公式、单位、换算或传播计算错误 |

## 5. 评分量规

建议总分 100 分，并保留分项分。

| 分项 | 建议分值 | 评分关注点 |
| --- | ---: | --- |
| 证据扎根 | 25 | 是否有明确证据、是否支持关键结论 |
| 标准适用性 | 20 | 标准名称、版本、适用行业、适用污染物是否正确 |
| 审核逻辑 | 20 | 判断链是否完整，是否说明前提和边界 |
| 数值/计算 | 15 | 限值、单位、公式、系数和结果是否可靠；无计算题可转入逻辑分 |
| 修改意见可执行性 | 15 | 是否能转化为报告修改要求 |
| 风险提示 | 5 | 是否标注证据不足、需专家复核、需补充样本等 |

分档解释：

| 分数段 | 含义 | 使用建议 |
| --- | --- | --- |
| 90-100 | 高质量候选答案，事实、依据、逻辑和修改建议完整 | 可进入专家确认 |
| 70-89 | 基本可用，但有轻微遗漏或证据表述不足 | 需人工补证据或润色 |
| 50-69 | 部分正确但存在明显缺口 | 不宜直接使用 |
| 30-49 | 严重错误或关键链条缺失 | 作为错误案例分析 |
| 10-29 | 大部分无关、错误或无证据 | 剔除 |

## 6. A/B/C/D 知识条件对照建议

可将 Chen 2026 的模型/agent 对比思路迁移为知识条件对照，而不是模型微调对照。

| 组别 | 输入条件 | 目的 |
| --- | --- | --- |
| A | 仅给审核问题和报告片段 | 观察无标准库条件下的基础判断 |
| B | A + 标准依据库检索片段 | 观察标准扎根是否改善 |
| C | B + 审核技能库步骤 | 观察结构化审核流程是否改善 |
| D | C + 类案经验/修改意见样本链 | 观察经验材料是否改善可执行修改意见 |

评价时必须分题型报告，不宜只报告总均分：

- L1/L2/L3 分层；
- 审核领域分层；
- 推理类型分层；
- 是否需要计算分层；
- 是否跨文档证据分层。

## 7. 统计验证建议

当人工评分样本形成后，可按以下顺序分析：

1. 对每组输出计算总分和分项分；
2. 用 Bootstrap 重采样计算 95% 置信区间；
3. 对同一题目的不同知识条件进行配对比较；
4. 若分布非正态，优先报告 Wilcoxon signed-rank 或与 paired t-test 并列；
5. 多组比较使用 FDR 校正；
6. 报告 Cohen's d 或其他效应量；
7. 配合错误案例分析解释分数差异。

中期阶段建议表述：

- 可以说“已形成可开展知识条件对照实验的评价框架”；
- 可以说“具备进一步人工评分和统计验证的基础”；
- 不宜说“已经证明技能库有效”。

## 8. 多评审者一致性

SI 的 Fleiss' kappa 结果显示，多评审者评分可能并不天然一致。本课题应注意：

1. 至少先用 10-20 道题进行评分口径校准；
2. 对专家评分不一致的题目记录分歧原因；
3. 若使用 AI 辅助评分，应与人工评分分开记录；
4. 不把 AI 评分等同于专家复核；
5. 后续可报告 Fleiss' kappa 或 Krippendorff's alpha 作为一致性指标。

## 9. 推荐落地顺序

1. 从现有 QA 中筛选 30-50 道有完整证据链的审核任务；
2. 按四维分类框架补标签；
3. 生成或整理 A/B/C/D 四组输出；
4. 用本文件评分量规进行双人以上人工评分；
5. 记录错误类型和拒收原因；
6. 形成小规模统计分析和典型案例；
7. 再决定是否扩大题量。
