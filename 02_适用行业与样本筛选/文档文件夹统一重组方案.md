# 文档文件夹统一重组方案

> **目标**：将受理公告、终稿、批复、修改意见、补正通知等散落文件，统一按项目组织到一个文件夹中。
> **当前问题**：文件分布在 5 个子目录下，命名不一致，难以按项目浏览完整的审批链。

---

**问题**：
1. 同一个项目的受理公告、批复、修改意见分散在 3-4 个不同子目录
2. 文件夹按"文件类型"分，不按"项目"分，无法一眼看到某项目的完整审批链
3. MinerU 解析结果和原始 PDF 分离，不利于对照
4. 新增项目时需要手动在多个目录创建文件

## 二、文件夹编号规则

```
SH_{编号}_{项目简称}/
  ├── 01_受理公告/          # 受理公告报告 + 受理通知书
  ├── 02_终稿_拟审批稿/     # 报批稿 / 拟审批稿 / 终稿
  ├── 03_批复/               # 批复文件 / 审批意见
  ├── 04_修改意见_补正通知/  # 技术审查修改意见 / 补正通知
  ├── 05_专家意见/           # 专家评审会意见（可选）
  └── 06_提取产物/           # 结构化提取结果
```

### 文件命名规范

```
{项目编号}_{文件类型}_{序号}.{扩展名}

示例：
SH_001_受理公告报告.pdf
SH_001_受理公告报告_mineru.md
SH_001_拟审批稿.pdf
SH_001_批复.pdf
SH_001_修改意见_01.docx
SH_001_补正通知_01.pdf
```

## 三、迁移脚本

### Step 1: 创建文件夹骨架

```python
import os

BASE = "05_样本链（按项目组织）"
os.makedirs(f"{BASE}/_元数据", exist_ok=True)

projects = [
    "SH_001", "SH_002", "SH_003", "SH_004", "SH_005",
    "SH_006", "SH_007", "SH_008", "SH_009", "SH_010",
    "SH_011", "SH_012",
    "AI_001", "AI_002",
]

for pid in projects:
    for sub in ["01_受理公告", "02_终稿_拟审批稿", "03_批复", 
                 "04_修改意见_补正通知", "05_专家意见", "06_提取产物"]:
        os.makedirs(f"{BASE}/{pid}/{sub}", exist_ok=True)
    
    # 创建项目信息文件
    with open(f"{BASE}/{pid}/_项目信息.md", "w") as f:
        f.write(f"# {pid}\n\n"
                f"- 项目名称：待填写\n"
                f"- 行业代码：待填写\n"
                f"- 建设单位：待填写\n"
                f"- 建设地点：佛山市顺德区\n"
                f"- 文件完整度：待填写\n")
```

## 五、迁移后备忘

### 旧目录处理

确认所有文件迁移成功后：
1. 旧 5 个子目录移到 `_归档_旧结构/` 下
2. CSV 元数据文件移到 `_元数据/` 下
3. 保留 `99_原始文件路径索引/` 中的路径映射记录

### 后续维护规则

- **新增项目**：在 `05_样本链（按项目组织）/` 下创建新的 `SH_{编号}/` 文件夹，按 01-06 子目录填充
- **新增文件**：按类型放入对应子目录，更新项目 `_项目信息.md`
- **MinUerU 解析**：原始 PDF 放入 `0X_文件类型/` 同级，加 `_mineru.md` 后缀
- **Git 提交**：每次新增或更新项目文件后，更新 `_元数据/sample_chain_inventory.csv`
