# Chen 2026 论文思路与本课题迁移方案

## 1. 研究问题层面的借鉴

Chen 等（2026）关注的是环境领域 LLM 的结构化数据构建和部署策略。对本课题而言，最可迁移的不是微调模型，而是以下方法论：

1. 环境领域知识需要被结构化、证据化和可检索化；
2. 复杂环境任务不能只依赖单轮问答，需要检索、工具、工作流和人工复核；
3. 生成式数据必须经过专家验证，未验证内容不能直接作为金标；
4. 模型/工作流评价需要题型分类、评分量规、统计分析和错误案例支撑。

本课题可将其迁移为：

`塑胶行业标准依据库 -> 审核技能库 -> 检索增强 -> 审核任务样本 -> 人工复核 -> 知识条件对照实验`

## 2. 数据构建思路

论文的数据构建流程可概括为：

1. 汇集权威教材语料；
2. 对教材进行 OCR、切块、向量化，构建向量数据库；
3. 同步构建全文检索库，支持关键词检索；
4. 使用种子问题在向量库和全文库中混合召回片段；
5. 去重并扩展上下文；
6. 使用 LLM 按固定 schema 生成结构化问题、答案和推理说明；
7. 由专家核对生成答案是否被原文证据支持；
8. 删除幻觉、证据不匹配或题文不一致的条目。

对本课题的迁移：

| Chen 2026 环节 | 本课题对应环节 |
| --- | --- |
| 教材语料 | 佛山塑胶指南、法规标准、Dify 知识库、环评报告、修改意见、批复 |
| 教材 chunk | 标准条目、报告片段、样本链片段、技能库步骤 |
| 种子问题 | 典型审核问题，如行业分类、废气收集效率、活性炭参数、危废识别 |
| 结构化问答 | 审核任务、参考答案、证据、修改意见、风险标签 |
| 专家验证 | 人工复核、专家校准、金标答案形成 |

## 3. SI 对数据验证的补充

SI 明确说明，训练数据生成后经过 expert-in-the-loop 验证。每条记录以统一格式呈现：

- model-generated question；
- model answer；
- model reasoning trace；
- retrieved textbook chunk。

专家主要检查：

1. factual alignment：答案中的数值、概念和技术描述是否能由来源片段支撑；
2. contextual relevance：问题是否与检索片段实质相关。

SI 报告的结果为：

- 生成 19,739 条；
- 保留 19,532 条；
- 剔除 207 条；
- 剔除条目不修正后重新纳入，因为问题多为结构性错误。

对本课题的要求：

- `05_QA测试集/` 目前应定位为种子问答或候选审核任务；
- 只有经过人工证据核验和专家校准的条目，才可作为正式评价题；
- reasoning trace 可用于复核透明度，但不能替代标准、报告或修改意见原文；
- 剔除条目应保留失败原因，便于论文中的错误分析章节使用。

## 4. SI 的失败样本类型

SI 将剔除样本分为三类：

| 类型 | 数量 | 本课题对应风险 |
| --- | ---: | --- |
| Null / vacuous responses | 144 | 检索片段只有标题、目录、术语表，无实质审核依据 |
| Source-question mismatch | 9 | 问题要求参数判断，但证据没有参数或适用条件 |
| Hallucinated / unsupported claims | 54 | 编造标准限值、产污系数、工艺参数或缩写 |

本课题建议扩展为以下错误标签：

- `vacuous_source`
- `question_source_mismatch`
- `unsupported_numeric_claim`
- `unsupported_standard_claim`
- `contextual_misinterpretation`
- `case_overgeneralization`
- `reasoning_chain_error`
- `numeric_propagation_error`

## 5. 测试集设计的迁移

SI 的 328 道测试题采用四维分类框架。本课题可直接借鉴这种“先分类、再评分”的设计。

### 5.1 认知层级

| SI 分类 | 数量 | 本课题迁移 |
| --- | ---: | --- |
| L1 Understanding | 104 | 标准、术语、行业边界解释 |
| L2 Application | 145 | 标准适用、工艺判断、数值核算 |
| L3 Analysis | 79 | 多材料冲突、跨标准综合、修改意见闭合判断 |

### 5.2 审核领域

本课题可按以下领域标注：

- 行业分类与适用边界；
- 三线一单与空间准入；
- 建设内容完整性；
- 环境质量与执行标准；
- 污染物排放标准；
- 产污系数与源强核算；
- 废气收集与治理；
- 活性炭参数；
- 固废与危废识别；
- 总量控制与管理要求；
- 样本链一致性。

### 5.3 推理类型

- factual/descriptive：抽取事实、解释概念；
- mechanistic：解释污染产生、收集、治理、失效机理；
- quantitative：限值、风量、源强、总量、危废量计算；
- comparative：比较标准、治理方案或报告版本；
- cross-document synthesis：跨标准库、报告、修改意见、批复和类案经验综合判断。

### 5.4 功能能力

- knowledge grounding：依据是否正确；
- numerical fidelity：数值和单位是否可靠；
- logical reasoning：审核链条是否完整；
- evidence integration：多来源证据是否整合；
- review comment actionability：修改意见是否可执行。

## 6. 评价与统计验证

SI 的评分提示词采用 10-100 分分档，关注事实正确性、逻辑推理、计算过程和完整性。迁移到本课题时，建议拆成分项分：

- 证据扎根；
- 标准适用性；
- 审核逻辑；
- 数值/计算；
- 修改意见可执行性；
- 风险提示。

SI 的统计验证包括：

- Shapiro-Wilk 正态性检验；
- Bootstrap 95% 置信区间，SI 使用 10,000 次重采样；
- paired t-test；
- Wilcoxon signed-rank；
- FDR 多重比较校正；
- Cohen's d 效应量。

本课题后续 A/B/C/D 知识条件对照不宜只报告平均分。更稳妥的报告方式是：

1. 总分和分项分；
2. 按题型分层的表现；
3. 置信区间；
4. 成对比较和效应量；
5. 典型错误案例。

## 7. 部署策略层面的迁移

Chen 2026 区分了 fine-tuned model、base model 和 agentic workflow 的表现。对本课题更有价值的是“分层部署”思想：

| 任务类型 | 推荐方式 |
| --- | --- |
| 稳定、规则明确、标准化任务 | 沉淀为标准库条目和审核 skill |
| 需要跨材料检索的任务 | 使用 RAG 和混合检索 |
| 需要计算的任务 | 接入公式、表格、脚本或人工复算 |
| 需要专业判断的任务 | 保留专家复核 |
| 需要形成修改意见的任务 | 结合样本链和类案经验，但标注风险 |

## 8. 不宜直接照搬的部分

- 不宜把论文教材 QA 直接作为环评审核数据；
- 不宜把 fine-tuning 作为当前项目主线；
- 不宜把外部模型比较结论直接用于本课题；
- 不宜把 AI 评分替代专家复核；
- 不宜把类案经验泛化为强制性标准。

## 9. 可写入论文的方法章节思路

本课题可在论文中将 Chen 2026 作为方法借鉴，表述为：

> 参考环境领域 LLM 结构化数据构建与部署研究，本文不直接开展模型微调，而是将其“权威材料结构化、检索增强、专家验证、分层评价”的方法迁移到塑胶行业环评审核场景，构建标准依据库、审核技能库和知识条件对照评价框架。
