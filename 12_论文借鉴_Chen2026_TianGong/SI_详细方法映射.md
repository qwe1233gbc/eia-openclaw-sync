## 2. 训练数据验证流程的可迁移点

SI 中的训练数据验证不是“生成后直接使用”，而是专家参与的闭环筛选。每条 AI 生成问答以标准化格式呈现给专家，包括：

1. model-generated question；
2. model answer；
3. model reasoning trace，仅用于提升复核透明度，不进入训练数据；
4. retrieved textbook chunk，作为事实来源。

专家主要检查两类问题：

| 复核维度 | SI 中含义 | 本课题迁移表达 |
| --- | --- | --- |
| Factual alignment | 答案中的数值、概念、技术描述必须能回溯到教材片段 | 审核结论、标准限值、工艺参数、修改意见必须能回溯到指南、标准、报告原文或修改意见 |
| Contextual relevance | 问题必须与检索片段存在实质关联 | 审核问题必须能由对应项目材料或标准条目支撑，不能用空泛片段生成“看似专业”的题目 |

SI 给出的规模与筛选结果：

- 自动流程生成 19,739 条记录；
- 专家验证后保留 19,532 条；
- 剔除 207 条；
- 剔除条目没有修正后重新纳入，因为问题通常是结构性错误，而不是局部可修补错误。

对本课题的启发：

- `05_QA测试集/` 中的种子问答应区分“候选问题”“已人工复核问题”“专家确认问题”；
- 证据不足、题文不匹配、数值无来源的条目应进入 rejected log，而不是简单改写后当作金标；
- reasoning 或中间推理可以作为复核辅助材料，但不应作为最终证据本身。

## 3. 剔除数据的三类失败模式

SI 将被剔除数据分为三类，并给出计数：

| SI 失败模式 | 计数 | 含义 | 本课题对应风险 |
| --- | ---: | --- | --- |
| Null / vacuous responses | 144 | 检索片段只有术语表、索引、标题等，模型回答“无具体数据”或生成低价值答案 | 标准库片段只有目录、表名或标题，无法支撑具体审核结论 |
| Source-question mismatch | 9 | 问题要求详细参数，但来源片段只有标题、概述或索引 | QA 问题要求判断活性炭参数、风量、VOCs 总量，但检索证据没有对应数据 |
| Hallucinated / unsupported claims | 54 | 模型生成来源中不存在的数值、缩写或技术判断 | 模型编造排放限值、产污系数、收集效率、行业分类依据 |

本课题建议采用同样的剔除标签：

- `vacuous_source`：证据片段无实质内容；
- `question_source_mismatch`：问题和证据不匹配；
- `unsupported_numeric_claim`：数值或限值无来源；
- `unsupported_standard_claim`：标准适用性判断无来源；
- `case_overgeneralization`：把类案经验泛化成强制标准；
- `calculation_chain_error`：计算链或单位换算错误。

## 5. 评价提示词与评分量规

SI 的评分提示词采用 10-100 分，并按 5 个档位区分：

| 分数段 | SI 评分含义 | 本课题迁移 |
| --- | --- | --- |
| 90-100 | 事实正确、逻辑严密、计算完整，覆盖参考答案 | 可作为高质量审核意见，但仍需人工确认 |
| 70-89 | 基本正确，有小遗漏或轻微错误 | 可作为候选答案，需要补证据或补表述 |
| 50-69 | 存在明显错误或关键步骤不足 | 不宜作为正式答案，需要人工重写或重新检索 |
| 30-49 | 严重错误、逻辑偏离、计算混乱 | 应判为失败输出 |
| 10-29 | 大部分错误或无关 | 应剔除并记录错误类型 |

本课题不建议只给总分。建议同时输出：

- `score_total`；
- `score_evidence_grounding`；
- `score_standard_applicability`；
- `score_calculation_or_logic`；
- `score_review_comment_actionability`；
- `error_tags`；
- `manual_review_required`。

## 6. 统计验证设计

SI 使用多种统计验证方法，而不是只报告平均分：

| 方法 | SI 用途 | 本课题迁移 |
| --- | --- | --- |
| Shapiro-Wilk normality test | 检查分数分布是否正态；SI 中各模型均非正态 | 后续对照实验先检查分布，不默认使用参数检验 |
| Bootstrap CI | 10,000 次重采样，估计平均分 95% 置信区间 | 小样本阶段也可用 Bootstrap 给出不确定性范围 |
| Paired t-test | 比较同一测试题上不同模型/工作流的均值差异 | 用于 A/B/C/D 知识条件对照的成对比较 |
| Wilcoxon signed-rank | 非参数稳健性检验 | 当分布非正态时，与 t-test 并行报告 |
| FDR correction | 多重比较校正 | 多组对照时避免误报显著性 |
| Cohen's d | 判断实际效应大小 | 避免只看 p 值，需说明改进是否有实际意义 |

对本课题的底线要求：

- 在样本量较小、金标未稳定前，不宜宣称“已经证明有效”；
- 可以表述为“形成了可开展对照实验的评价框架”；
- 若后续进行 A/B/C/D 对照，应报告平均分、置信区间、题型分层表现和典型错误，而不是只报告一个总分。

## 7. 代表性响应对比的启发

SI 第 14-18 页展示多类问题下不同模型输出的对比。可迁移的观察维度包括：

- 结构化与层级化表达；
- 术语和领域原则准确性；
- 因果链和工程推理完整性；
- 是否整合实际工况、环境因素和管理要求；
- 是否有详细事实描述但不过度堆砌；
- 是否在比较分析中说明适用场景和边界条件。

本课题可将这些维度迁移为审核意见质量诊断：

| SI 观察维度 | 环评审核输出要求 |
| --- | --- |
| Structural organization | 审核意见应分为问题、依据、判断、修改建议 |
| Domain terminology | 正确使用 VOCs、NMHC、排气筒、无组织、危废代码等术语 |
| Causal reasoning | 说明为什么某工艺、风量、效率或标准适用/不适用 |
| Practical factors | 结合项目工况、原辅料、设备、治理设施和管理要求 |
| Comparative analysis | 对多个标准、多个版本报告或多个治理方案进行边界比较 |

## 8. 错误案例分析框架

SI 第 19 页的代表性错误案例可迁移为本课题错误标签：

| SI 错误类型 | 本课题典型表现 | 建议处理 |
| --- | --- | --- |
| Hallucination | 编造标准限值、污染物、工艺参数或批复要求 | 直接判为不合格，要求回查原文 |
| Factual drift | 标准名称正确但适用条件、污染物或限值说偏 | 标记标准适用性错误 |
| Contextual misinterpretation | 把普通注塑项目误判为涉胶水/复合项目，或反之 | 回到行业边界和项目原文 |
| Terminology inaccuracy | C2929、C2926、NMHC、TVOC、VOCs 混用 | 进入术语校准清单 |
| Reasoning-chain error | 判断链跳过关键前提，如未核实集气形式就判断效率 | 要求补充前提证据 |
| Numeric propagation | 单位换算、风量、总量、活性炭更换周期计算传播错误 | 标记 calculation_needed 并人工复算 |

## 9. Fleiss' kappa 的启发

SI 第 20 页报告多评审者一致性：

- 所有 5 个评价者的 Fleiss' kappa 约为 0.22；
- 3 个评价者组合中最高一致性组约为 0.424，超过 0.40 阈值；
- 4 个评价者组合最高约为 0.263，仍低于 0.40 阈值。

这说明：

- 多模型或多评审者评分本身可能存在较大不一致；
- 需要先校准评分标准，再扩大评价规模；
- 本课题若邀请专家复核，应记录专家一致性，而不是默认单一专家意见绝对稳定；
- 若使用 AI 辅助评分，应将其定位为辅助筛查，不作为最终专家判断。

## 10. 对本课题最稳妥的迁移结论

可直接借鉴：

1. 生成条目必须绑定检索证据；
2. 人工复核必须同时检查事实扎根和问题-证据匹配；
3. 被剔除条目要记录失败类型；
4. 测试题应按认知层级、领域、推理类型、功能能力分类；
5. 评分应分档，并记录错误标签；
6. 对照实验应报告统计不确定性和错误案例；
7. 多评审者一致性需要校准。

不宜直接照搬：

1. 不宜把教材 QA 直接当成本课题数据；
2. 不宜把 fine-tuning 作为当前研究重点；
3. 不宜把 SI 中模型优劣结论直接迁移到环评审核任务；
4. 不宜把 AI 评分当作专家复核替代。
