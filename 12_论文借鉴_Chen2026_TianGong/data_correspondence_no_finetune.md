# 本项目数据与 Chen 2026 方法的对应关系（不微调版）

> 结论先行：本课题的数据与 Chen 2026 可以形成较清晰的方法对应，但对应的是“权威材料结构化、混合检索、工作流化任务执行、人工复核评价”这一条线，而不是“训练数据构建后进行模型微调”这一条线。

## 1. 总体判断

本项目目前已经具备与 Chen 2026 进行方法类比的基础，主要体现在：

1. 已有较明确的领域语料来源：佛山塑胶指南、Dify 知识库、标准库、样本链、修改意见和审核技能库；
2. 已将部分标准、流程和审核经验拆解为可检索、可调用的结构化材料；
3. 已形成种子审核问题和知识条件对照实验思路；
4. 已具备把“模型回答质量”转化为“证据扎根、标准适用、审核逻辑、修改意见可执行性”等维度评价的条件。

但需要明确：本项目当前不做微调，也不需要把 Chen 的 fine-tuning 数据集路线完整照搬。更稳妥的论文定位是：

> 参考 Chen 2026 在环境领域中对权威材料结构化、检索增强、工作流部署和专家验证的思路，构建面向塑胶行业环评审核的标准依据库、审核技能库和知识条件对照评价框架。

## 3. 本项目现有数据状态

### 3.1 标准依据库

当前可对应 Chen 的“权威语料结构化”部分。

- `03_指南解析_明文标准库/standard_cards.jsonl`：当前约 37 条结构化标准卡；
- `03_指南解析_明文标准库/plastic_guide_standard_library_v3_deep.jsonl`：当前约 71 条较早版本标准条目；
- `cards_by_skill/`：已按审核 skill 拆分标准依据；
- `Dify工作流知识库/`：包含国民经济行业分类、三线一单、排放标准、产污系数、废气收集、危废和 VOCs 管理等材料。

现阶段适合用于检索增强和技能库调用，但仍需要继续校准标准版本、适用条件、强制性边界和人工复核状态。

### 3.2 审核技能库

当前可对应 Chen 的“agentic workflow / deployment strategy”部分。

- `09_环评审核技能库/` 已形成 15 个独立审核 skill 和 1 个总审核汇总 skill；
- 每个 skill 负责一个较清晰的审核任务，例如行业类别、三线一单、建设内容、排放标准、产污系数、废气收集、活性炭参数、危废识别、VOCs 总量控制；
- `skill_registry.yaml` 已记录 skill 与 Dify 工作流来源的对应关系。

这部分是本项目最适合类比 Chen 的地方：不是训练一个新模型，而是把稳定审核规则和工作流拆成可调用的技能模块。

### 3.3 样本链材料

当前可对应 Chen 的“证据来源和人工验证材料”部分，但完整度仍需补强。

- `05_样本链（按项目组织）/` 下已有按项目组织的报告、终稿、修改意见、批复等材料；
- 当前项目目录约 33 个，包含正式样本、候选样本、排除样本和复核样本；
- `_元数据/sample_chain_inventory.csv` 中已有小规模样本链清单，用于标注是否可作为输入文本、参考答案和真实错误标签。

这部分不宜直接表述为“完整金标数据集”。更稳妥的说法是：已形成样本链整理框架和部分可用证据，但仍需补全涉胶水、复合、熟化、VOCs 治理等典型项目的完整链条。

### 3.4 QA 与审核任务

当前可对应 Chen 的“种子问题和测试题分类”部分，但还不能等同于正式测试集。

- `05_QA测试集/seed_qa_942题_151项目_phase1+phase2合并.xlsx` 当前约 942 条种子问答；
- 字段包括项目、问题、答案、证据、来源依据、可验证性、人工复核等；
- 其中部分人工审核列存在编码显示问题，后续应统一字段名并清理乱码列。

这些内容适合先作为候选审核任务池。只有完成证据核验、人工复核和金标答案校准后，才能进入正式对照实验。

### 3.5 知识条件对照实验

当前可对应 Chen 的“模型/策略比较”，但不采用微调对照。

推荐对照方式为：

| 组别 | 输入条件 | 评价目的 |
| --- | --- | --- |
| A | 仅报告片段或审核问题 | 观察模型基础判断能力 |
| B | A + 标准依据库检索结果 | 观察标准依据是否提高事实扎根 |
| C | B + 审核技能库步骤 | 观察结构化审核流程是否提高逻辑完整性 |
| D | C + 样本链/类案经验/修改意见片段 | 观察真实经验材料是否提高修改意见可执行性 |

这里的核心创新点是“知识条件对照”，不是“微调模型对照”。

## 4. 不微调时，本项目可以怎样类比 Chen

本项目可以保留 Chen 的方法骨架，但替换实现对象：

```text
Chen 2026:
权威教材 -> 文档解析/切块 -> 检索 -> 生成结构化 QA -> 专家验证 -> 模型/工作流评价

本项目:
塑胶指南/标准/样本链/修改意见 -> 标准卡与报告片段 -> 混合检索 -> 审核任务与候选意见 -> 人工复核 -> 知识条件对照评价
```

因此，论文中可以重点强调以下迁移：

1. **语料对象迁移**：从环境教材迁移到塑胶行业环评审核材料；
2. **任务对象迁移**：从环境知识问答迁移到环评报告审核意见生成；
3. **组织方式迁移**：从教材 chunk 迁移到标准卡、技能步骤和样本链证据；
4. **验证方式迁移**：从专家验证 QA 迁移到人工复核审核意见和修改依据；
5. **评价方式迁移**：从模型能力比较迁移到不同知识条件下审核辅助质量比较。

## 5. 不建议写成的方向

为避免论文表述过度，建议不要写：

- 本研究复现了 Chen 2026 的完整训练流程；
- 本研究构建了可用于微调的最终训练数据集；
- 本研究证明技能库或 RAG 系统已经优于人工审核；
- 本研究实现了环评自动审批或自动审查；
- TianGong_Env 数据可以直接作为本课题环评审核金标。

## 6. 推荐论文表述

可以写成：

> 本研究不开展模型微调，而是参考 Chen 2026 在环境领域大模型研究中提出的结构化知识构建、检索增强、工作流部署和专家验证思路，将其迁移到塑胶行业环评审核场景。研究以佛山塑胶指南、标准规范、样本链和修改意见为主要语料，构建标准依据库和模块化审核技能库，并设计 A/B/C/D 知识条件对照实验，用于评估不同知识输入对审核意见生成质量的影响。

更简洁的中期汇报表述可以写成：

> 本课题与 Chen 2026 的对应关系主要体现在方法层面：借鉴其“材料结构化、检索增强、工作流化、专家验证和分层评价”的研究思路，但不进行模型微调；当前成果仍属于阶段性数据组织和评价框架建设，后续需继续补充真实样本链、专家复核和金标答案校准。

## 7. 当前缺口

当前还需要补强的部分包括：

1. 完整样本链数量不足，尤其是涉胶水、涂胶、复合、熟化和 VOCs 治理典型项目；
2. QA 题库中部分字段和人工审核列仍需统一编码、字段名和复核状态；
3. 标准依据库需要继续校准来源、适用条件、版本有效性和强制性边界；
4. 类案经验仍需与真实修改意见、批注或技术审查意见逐条绑定；
5. A/B/C/D 对照实验还需要稳定金标答案、人工评分和一致性检验。

## 8. 下一步建议

优先顺序建议如下：

1. 从 942 条种子问答中筛选 30-50 条证据链较完整的题目；
2. 对每题补齐标准依据、报告片段、修改意见或批复来源；
3. 按 L1/L2/L3、审核领域、推理类型和功能能力进行标签化；
4. 用 A/B/C/D 四组知识条件生成候选审核意见；
5. 使用 `evaluation_record.schema.json` 记录人工评分和错误标签；
6. 只在人工复核完成后报告阶段性实验结果。
