# 佛山市塑胶行业环评智能审核研究材料包

> 更新时间：2026-06-29  
> 定位：本仓库不是全量环评资料库，而是面向“佛山市塑胶行业环评文件智能审核辅助”的研究材料包。  
> 当前阶段：研究对象、数据来源、标准依据库、类案经验库、审核任务样本和 A/B/C 知识条件对照实验已初步形成，仍需专家复核和样本链补强。

## 1. 当前研究方向

本课题当前研究方向是：

**面向佛山市塑胶行业建设项目环评文件，构建证据可追溯的标准依据库、样本证据链和类案经验库，并评估不同知识条件对大模型审核辅助质量的影响。**

研究对象已经从普通 C2929 注塑项目，收敛到《佛山市塑胶行业建设项目环评文件编制技术参考指南（试行）》适用范围内的塑胶制造项目，重点关注：

- 涉胶水、涂胶、贴合、复合、熟化等工艺；
- 塑料薄膜、塑料包装材料、塑塑复合、塑铝复合等场景；
- VOCs 产生、收集、治理、总量控制和活性炭管理等审核问题；
- 与佛山塑胶指南、排放标准、产污系数、三线一单和国民经济行业分类相关的审核依据。

普通 C2929 纯注塑项目如果不涉胶水、复合或典型 VOCs 治理环节，原则上只能作为参照样本，不宜作为指南适用性的核心验证对象。

## 2. 本课题不做什么

为避免研究边界泛化，本仓库暂不定位为：

- 通用环保问答数据集；
- 全行业环评自动审核系统；
- 自动审批或替代人工审批工具；
- 直接以微调模型为核心的最终系统；
- 对所有 C2929 注塑项目一概适用的审核规则库。

当前更稳妥的技术路线是：**知识库构建 + 混合检索 + 结构化审核任务 + 人工复核 + 对照实验**。只有当某类审核任务稳定、样本链完整、金标答案经过专家校准后，才适合进一步讨论微调或自动化部署。

## 4. 标准库与经验库的分工

| 类型 | 来源 | 作用 | 当前使用边界 |
| --- | --- | --- | --- |
| 标准库 / 明文依据库 | 佛山塑胶指南、国家报告表指南、排放标准、技术规范、产污系数、三线一单等 | 告诉模型“依据是什么、报告应提供什么证据、限值和公式是什么” | 已具备阶段性使用条件，但仍需逐条校准来源、适用条件和版本有效性 |
| 经验库 / 类案经验库 | 顺德已审报告、修改意见、真实批注、同类项目审核经验 | 告诉模型“类似项目实际容易错在哪里、经办人可能追问什么、需要补什么证据” | 仅能在适用性过滤后使用；未绑定真实证据的条目只能作为候选经验 |

经验库不能替代标准依据。经验条目必须说明适用行业、工艺条件、任务类型和证据来源，避免跨行业、跨工艺或跨任务类型的过度泛化。

## 5. 样本链材料怎么用

| 文件类型 | 角色 | 用途 |
| --- | --- | --- |
| 受理公告报告 | 待审核文本 / 输入材料 | 用于模拟模型需要审核的原始报告，不直接作为正确答案 |
| 修改意见 / 技术审查意见 | 真实问题标签 | 用于识别报告中实际被指出的问题，是经验库和审核任务的重要来源 |
| 终稿 / 拟审批稿 | 修改后参考版本 | 用于观察问题是否被修正，并辅助校准参考答案 |
| 批复文件 | 审批结论和管理要求 | 用于提取正式审批条件、排放要求和管理约束 |

样本按资料完整度分级：

| 样本等级 | 定义 | 允许用途 |
| --- | --- | --- |
| A_complete_chain | 同一项目同时具备受理公告报告、修改意见或技术审查意见、终稿 / 拟审批稿、批复文件 | 贯穿式案例、审核前后对照、错误修正路径分析，优先进入正式评价任务 |
| B_partial_chain | 同一项目具备其中两类或三类材料 | 经验条目、证据片段、种子 QA；人工复核后可进入评价任务 |
| C_single_source | 只有单个文件来源，例如只有报告、批复、修改意见或标准依据 | 标准库、经验库候选条目、规则说明；原则上不直接进入正式评价任务 |

缺少 `evidence`、`source_basis` 或人工复核结论的 QA，不应直接作为金标答案。无法追溯来源的经验条目只能保留为 candidate，不能进入 active 使用。

当前已经阶段性形成以下基础：
