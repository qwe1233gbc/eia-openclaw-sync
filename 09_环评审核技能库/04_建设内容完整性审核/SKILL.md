# 04_建设内容完整性审核

## 1. 审核目标
建设内容完整性审核的独立审核，判断报告中的相关内容是否满足法规、标准和指南要求。

## 2. 适用场景
- 佛山市塑胶行业建设项目环境影响报告表
- 涉VOCs、胶水/涂胶、复合/熟化工艺的项目

## 3. 不适用场景
- 报告书级别环评文件
- 非塑胶行业项目
- 纯注塑/挤塑项目（无VOCs产生环节，参照使用）

## 4. 输入材料
- 环评报告MinerU解析后的Markdown全文
- 可选：法规库检索结果（从 `03_法规库_明文依据/` 获取）
- 对应Dify工作流：`06_Dify工作流/5-建设内容编制完整性判断.yml`

## 5. 文本提取策略

### 关键词评分筛选
使用全文检索，无预定义关键词权重

### 章节边界定位
- 从报告全文检索相关章节

原始工作流使用关键词评分累积 + 章节边界截断的混合策略，总分超过阈值且位于目标章节内的文本片段被提取为审核素材。

## 6. 法规库检索

审核时需从法规库检索以下模块的标准条款：

| STD_007 | 项目组成完整性 | 环办环评[2020]33号 附件2 | 须列表说明主体/辅助/公用/环保/储运/依托工程，注明建筑面积、层数、建筑功能 [依据: 环办环评[2020]33号 附件2] |
| STD_008 | 产品方案 | 环办环评[2020]33号 附件2 | 产品方案须含序号、产品名称、设计产能、规格尺寸。涉涂装/复合/印刷工序须注明涂装面积、复合面积或印刷面积 [依据: 环办环评[2020]33号 附件2] |
| STD_009 | 改扩建项目依托关系 | 环办环评[2020]33号 附件2 | 改扩建/迁建/技改项目须说明与现有工程的依托关系（公辅设施/环保设施/生产设备），注明'以新带老'内容 [依据: 环办环评[2020]33号 附件2] |
| STD_020 | 工艺流程图 | 环办环评[2020]33号 附件2 | 工艺流程图须标注所有产污环节（G废气/W废水/N噪声/S固废），注塑/挤出/涂布/复合/印刷/贴合/烘干/熟化各工序单独标注。图须与设备清单一一对应 [依据: 环办环评[2020]33号 附件2] |
| STD_068 | 附图要求 | 环办环评[2020]33号 附件2 | 须附:总平面布置图/设备布置图/雨污管网图/监测点位图/四至图/现状照片。图纸含比例尺/指北针/图例 [依据: 环办环评[2020]33号 附件2] |

## 7. 审核判断清单

| 审核点 | 触发条件 | 所需证据 |
|--------|---------|---------|
| 项目组成完整性 | 项目组成表不完整 | 项目组成一览表(表3格式), 总平面布置图 |
| 产品方案 | 产品方案表缺少关键参数 | 产品方案一览表(表2格式), 产品规格说明书 |
| 改扩建项目依托关系 | 改扩建项目未说明与现有工程关系 | 现有工程环评批复及验收文件, 依托可行性分析, 以新带老工程内容表 |
| 工艺流程图 | 工艺流程图缺产污标注或与设备清单不对应 | 工艺流程图(含产污标识), 设备清单与流程图的对应表 |
| 附图要求 | 附图缺比例尺/指北针/图例 | 各图纸及图件清单, 图纸与正文一致性核对表 |

## 8. 判断依据与限值

- **项目组成完整性**: 须列表说明主体/辅助/公用/环保/储运/依托工程，注明建筑面积、层数、建筑功能 [依据: 环办环评[2020]33号 附件2]
- **产品方案**: 产品方案须含序号、产品名称、设计产能、规格尺寸。涉涂装/复合/印刷工序须注明涂装面积、复合面积或印刷面积 [依据: 环办环评[2020]33号 附件2]
- **改扩建项目依托关系**: 改扩建/迁建/技改项目须说明与现有工程的依托关系（公辅设施/环保设施/生产设备），注明'以新带老'内容 [依据: 环办环评[2020]33号 附件2]
- **工艺流程图**: 工艺流程图须标注所有产污环节（G废气/W废水/N噪声/S固废），注塑/挤出/涂布/复合/印刷/贴合/烘干/熟化各工序单独标注。图须与设备清单一一对应 [依据: 环办环评[2020]33号 附件2]
- **附图要求**: 须附:总平面布置图/设备布置图/雨污管网图/监测点位图/四至图/现状照片。图纸含比例尺/指北针/图例 [依据: 环办环评[2020]33号 附件2]

## 9. 输出格式

```json
{
  "skill_id": "04",
  "skill_name": "04_建设内容完整性审核",
  "conclusion": "匹配 | 不匹配 | 部分匹配 | 无法判断",
  "key_findings": [
    {
      "review_point": "",
      "finding": "",
      "evidence_location": "",
      "law_basis": ""
    }
  ],
  "evidence_used": [],
  "law_basis_used": [],
  "review_comment": "",
  "manual_review_needed": true
}
```

### 输出说明
- `conclusion` 只能是四类：匹配/不匹配/部分匹配/无法判断
- `review_comment` 须写成经办人可直接执行的审核意见，引用具体条款号和限值
- `manual_review_needed` 默认 `true`，仅当所有审核点均"匹配"且证据链完整时可为 `false`
- 没有证据时必须输出"无法判断"，禁止编造审核结论

## 10. 人工复核提示

- 所有 `conclusion` 为"不匹配"或"无法判断"的条目 **必须** 人工复核
- 证据来源必须可追溯到报告原文的章节/页码
- 法规依据如有更新（如标准修订、地方政策调整），需同步更新本 skill 的判断阈值
- 若报告缺失关键信息（如无MSDS、无设备参数），标记为"无法判断"并指明缺失项
