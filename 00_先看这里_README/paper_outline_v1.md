# 面向环评智能审核的结构化知识库构建与对照实验验证

> 论文框架 v1 | 基于 eia-openclaw-sync 仓库全部研究产出
> 数据截止: 2026-06-23

---

## 摘要

环境影响评价（EIA）审批面临报告数量大、审核标准多、经验依赖强的三重困境。大语言模型（LLM）为辅助审核提供了可能，但单纯依赖模型存在知识幻觉和依据缺失的局限。本文提出"明文标准库+类案经验库"双库驱动的环评审核框架：标准库从26个原生标准文件中按71个审核点结构化抽取条款；经验库从902条真实审批意见中提炼18条可复用的审核模式。基于佛山市顺德区26份塑胶行业环评报告，通过A/B/C三组知识条件对照实验（仅报告/报告+标准库/报告+标准+经验库）验证双库框架的增量作用。

## 一、引言

### 1.1 研究背景

- 环评审批现状：顺德区年审批量11万+条，人工审核效率瓶颈
- 大模型在垂直领域的机遇与挑战：幻觉问题、缺乏领域知识、无法引用具体标准条款
- 知识库+LLM的技术路线：RAG（检索增强生成）框架下结构化知识库的核心地位

### 1.2 核心问题

1. 如何从法规标准和审批实践中**系统构建**可引用的领域知识库？
2. 标准库和经验库能否**独立且叠加地**提升LLM的审核能力？
3. 如何**量化评估**知识库的增量作用？

### 1.3 研究范围

- 领域：佛山市塑胶行业（C292塑料制品业）环评报告表
- 审核焦点：VOCs废气收集+活性炭治理（涉胶水/涂布/复合工艺）
- 地域：佛山市顺德区

## 三、知识库构建方法

### 3.1 总体框架

```
                    ┌──────────────┐
                    │  环评报告输入  │
                    └──────┬───────┘
                           │
              ┌────────────┼────────────┐
              ▼            ▼            ▼
        ┌──────────┐ ┌──────────┐ ┌──────────┐
        │ 明文标准库 │ │ 类案经验库 │ │ LLM审核 │
        │ 71条标准卡 │ │ 18条经验卡 │ │ 16个工作流│
        │ 26个标准源 │ │ 902条QA源 │ │          │
        └──────────┘ └──────────┘ └──────────┘
              │            │            │
              └────────────┼────────────┘
                           ▼
                  ┌────────────────┐
                  │  结构化审核意见  │
                  └────────────────┘
```

### 3.2 明文标准库构建

**数据来源**：26个原生标准文件（8个GB/6个HJ/5个DB44/3个通知+指南）

**构建流程**：
1. 标准全文解析：MinerU → Markdown
2. 条款粒度拆分：按审核模块归入17个类别
3. 结构化字段：`id, module, requirement, source_file, source_section, standard_refs, trigger, check_evidence`

**关键指标**：
- 71条标准卡，每条包含原生标准引用（标准号+条款号+关键内容）
- 覆盖28个标准来源，最密集引用：HJ 2026-2013(10次)、环办环评[2020]33号(12次)、HJ 1122-2020(5次)
- 平均每条标准卡60-150字核心要求 + 1-3个标准引用

### 3.3 类案经验库构建

**数据来源**：649条审批QA + 253条严格QA + 113条技术审查意见 + 30份实际环评报告修改意见

**构建流程**：
1. 从902条QA中按关键词检索、去重、归类
2. 提炼为18条结构化经验卡
3. 五维度测试价值判断：来源真实性、问题具体性、证据可查性、增量作用、实验适用性

**经验卡结构**：
- `name, applicable_scope, when_triggered, what_to_check, case_reasoning, review_comment`
- `source_project, source_file, source_comment`（可追溯的审批来源）
- `evidence_level` (A/B/C) + `classification` (core/reference/pending)
- `applicable_modules` + `relevance_rules`（控制经验在实验中的触发条件）

**构成**：
- 高价值（适合对照实验）：5条（胶水VOCs、活性炭更换周期、收集效率、活性炭碘值、产污系数）
- 中价值（案例展示）：11条
- 暂不使用：2条

### 3.4 标准库与经验库的分工

| 维度 | 明文标准库 | 类案经验库 |
|------|-----------|-----------|
| 知识类型 | 合规性底线 | 实操高风险点 |
| 来源 | 法规标准原文 | 真实审批修改意见 |
| 作用 | "超不超标" | "哪里最容易藏问题" |
| 引用方式 | 标准号+条款号+原文 | 案例项目+审批意见摘要 |
| 触发条件 | 模块匹配 | 触发条件+模块+行业特征多层过滤 |

## 五、实验设计

### 5.1 A/B/C三组知识条件对照

| 组别 | 输入条件 | 符号 |
|------|---------|------|
| A组 | 仅环评报告原文 | Report-only |
| B组 | 报告 + 明文标准库 | +Standard |
| C组 | 报告 + 标准库 + 类案经验库 | +Standard+Experience |

### 5.2 评分维度（五维度量规）

| 维度 | 定义 | 评分标准 |
|------|------|---------|
| 问题识别 | 能否发现报告中的实质性问题 | 0=未体现 / 1=部分 / 2=准确 |
| 证据引用 | 审核依据是否可追溯到报告原文 | 0=无引用 / 1=笼统 / 2=精确到页码 |
| 逻辑完整性 | 问题→证据→标准→结论的链条是否完整 | 0=断裂 / 1=部分 / 2=完整 |
| 意见可执行性 | 修改建议是否具体、可操作 | 0=模糊 / 1=部分 / 2=可直接执行 |
| 误报控制 | 是否产生虚假或无依据的问题 | 0=严重误报 / 1=轻微 / 2=无误报 |

每组满分10分，5维度×2分

### 5.3 实验控制

- 经验库触发控制：仅当任务类型、审核模块和行业特征均匹配时触发
- 负对照：Q06题设计为"不触发任何经验"以验证经验库不会错误介入
- 数据泄漏防范：测试项目自身的修改意见不作为该项目的经验输入

### 5.4 Dify工作流实现

16个审核工作流覆盖12个审核节点：行业分类→投资核算→三线一单→建设内容完整性→环境质量→执行标准→排放标准→产污系数→废气收集形式→风量核算→收集效率判定

## 六、结果与分析

### 6.1 知识库构建结果

- 标准库：71条，100%标注原生标准引用，覆盖28个来源
- 经验库：18条，来自902条审批QA，覆盖9个审核模块
- Dify工作流：16个，覆盖12个审核节点
- 标准全文解析：26个标准/文件

### 6.2 预期结果（待实验验证）

**假设1**：B组（+标准库）相比A组（仅报告）在"证据引用"和"逻辑完整性"维度有显著提升。

**假设2**：C组（+标准+经验）相比B组在"问题识别"和"意见可执行性"维度有进一步提升，尤其体现在以下三个模块：
- 胶水VOCs含量（经验提示"超限值+需替代论证"，标准只给限值）
- 活性炭更换周期（经验提示"公式计算vs填报值差距"，标准只给公式）
- 收集效率取值（经验提示"半密闭取95%是高频错误"，标准只给效率表）

**假设3**：C组的"误报控制"不显著劣于B组，证明经验库不会引入大量误报。

### 6.3 经验库增量作用的边界

以下场景经验库**不**提供增量：
- 纯标准合规性判断（如"排放浓度是否达标"）——标准库已足够
- 不涉及审批实践模式的单次性问题（如"某个特定设备参数"）
- 来源缺失或证据等级为C的经验（如当前EXP_05/07）

## 七、讨论

### 7.1 双库互补机制

标准库保证"不漏审"，经验库保证"审得深"。两者的关系不是"A或B"而是"A且B"——只有标准没有经验，审核停留在合规性表层；只有经验没有标准，审核缺乏法定依据。

### 7.2 经验库的构建成本与复用性

- 当前18条经验来自902条QA的人工筛选，边际成本递减（每新增1条所需新QA标注量下降）
- 经验库的跨行业迁移：废气收集效率类经验（如"半密闭取95%"）对家具、印刷、涂装行业同样适用
- 经验库的时效性：政策变化（如顺德2:1替代→1:1替代）需定期更新

### 7.3 方法局限性

- 样本局限于顺德区+塑胶行业，泛化能力待验证
- A/B/C实验仅有18题的种子测试集，统计显著性有限
- 经验库的relevance_rules依赖人工定义，规模化需自动化匹配

---

## 八、结论

本文构建了面向环评智能审核的"明文标准库+类案经验库"双库框架，包含71条标准卡（26个标准源）和18条经验卡（902条审批QA来源）。基于26份顺德塑胶行业环评报告，设计了A/B/C三组知识条件对照实验（五维度评分量规），以量化验证双库的独立和叠加增量作用。双库框架为LLM在环评审核领域的可靠应用提供了可追溯的知识基础和可复用的评估体系。
