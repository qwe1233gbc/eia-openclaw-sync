# Related Work 高水平草稿 - Codex Draft 2026-06-16

## 2 相关研究

环境影响评价（Environmental Impact Assessment, EIA）是典型的知识密集型行政技术审查活动。环评文件审核不仅要求识别项目建设内容、生产工艺、污染源强、治理设施和环境风险等事实信息，还需要将报告文本与法律法规、行业指南、排放标准、技术规范、地方管理要求及既往审核经验相互对应。近年来，大语言模型（Large Language Models, LLMs）和检索增强生成（Retrieval-Augmented Generation, RAG）为长文本理解、法规问答和辅助决策提供了新的技术路径，但在环评审核场景中，单纯依赖通用模型或非结构化检索仍难以满足审核意见的依据可追溯性、经验适用边界和人机协同安全性要求。围绕本研究“标准--经验双库”的方法定位，相关研究可从环评智能化应用、环评知识组织、类案经验与智能体记忆、以及人机协同评价四个方面展开。

### 2.1 环评智能化与大语言模型辅助审核研究

已有研究开始探索人工智能在环评筛查、公众审查、许可问答和环境报告分析中的应用。Cilliers 等（2025）以南非环评筛查制度为对象，构建了多个版本的 GPT 辅助筛查工具，并在 20 个真实案例上测试其对适用活动清单和触发条件的识别能力。该研究表明，经过场景化定制和迭代优化的 GPT 工具能够在环评筛查阶段提供一定辅助价值，但在涉及多重触发条件和复杂建设内容时仍存在困难，因而强调人工监督和持续校准的重要性。与此相近，Meyur 等（2025）面向美国 NEPA 环境审查与许可场景提出 NEPAQuAD 数据集和 MAPLE 评估框架，以真实 EIS 文档构建环境许可问答 benchmark，比较了多种大模型在零样本、上下文驱动和 RAG 条件下的表现。其结果显示，环境审查任务具有较强的领域语义复杂性，模型在获得高质量证据片段时表现更好，而直接处理长篇 PDF 或依赖通用知识并不足以稳定支撑监管推理。

除筛查和问答外，Gimenez 等（2025）将开放数据、人工智能和影响显著性比较分析结合，用于增强公众对环评报告的技术审查能力。该研究并不将 AI 视为审批替代者，而是强调其在揭示数据质量、专家判断偏差、评价不确定性和方法不一致方面的辅助作用。这一取向与本研究的基本立场一致：环评智能审核应首先服务于证据发现、依据追踪和问题提示，而不是替代经办人作出最终行政判断。

上述研究共同表明，LLM 在环评领域的潜力已经从一般文本生成转向面向制度任务的场景化应用。然而，现有研究主要集中于“是否需要环评”的筛查问题、EIS/NEPA 文档问答、公众审查或环境报告信息识别，较少针对建设项目环评报告表的技术审核环节，尤其缺少将明文标准依据与地方类案审核经验分层组织并进行对照验证的研究。本研究所关注的佛山市塑胶行业建设项目审核，既需要模型识别报告中的 VOCs 源强、胶水使用、复合/熟化工艺、废气收集效率、活性炭治理、危废识别等技术问题，也需要明确每一项判断究竟来自标准条文、公式限值，还是来自同类项目中的经办人式追问经验。因此，仅将环评文件作为长文本问答对象是不充分的，仍需进一步构建可追溯、可过滤、可验证的审核知识体系。

### 2.2 环评知识组织、知识图谱与法规文本结构化研究

环评智能审核的前提是将分散在法规、导则、指南和报告文本中的知识转化为机器可调用的结构化形式。Garrido 和 Requena（2011）较早提出面向 EIA 的本体构建方法，指出不同国家、地区和方法体系中的环境评价术语存在异质性，若缺少统一概念框架，将难以支持知识共享、方法复用和智能系统开发。该研究通过构建 EIA ontology 和知识动员界面，为环评知识的概念化、术语统一和专家参与更新提供了基础思路。

更一般地，知识图谱研究为复杂领域知识的表示、查询、验证和抽取提供了理论与技术框架。Hogan 等（2022）系统总结了知识图谱的数据模型、查询语言、约束验证、知识抽取和推理机制；Schneider 等（2022）则从自然语言处理角度梳理了知识图谱在 NLP 任务中的应用谱系。这些研究说明，对于依赖多源异构知识的专业文本任务，结构化知识能够提升语义约束和可解释性。监管文本处理方面，Bommarito 等（2018）提出 LexNLP 工具包，用于法律和监管文本中的章节、日期、实体、数值和其他结构化信息抽取，说明规则性和制度性文本天然适合被转换为可计算的信息单元。

不过，从知识图谱或法规 NLP 到环评审核仍存在落差。传统知识图谱强调实体、关系和查询，而环评审核更关心“某一报告事实是否满足某一标准要求”“报告应补充何种证据”“某一计算公式或限值适用于哪个审核模块”。因此，本研究没有直接构建完整图数据库，而是采用“标准卡”的中间表示方式，将佛山市塑胶行业指南、国家报告表指南、VOCs 排放标准、危废识别规则、监测技术规范和地方政策要求转换为带有来源、适用模块、审核要点、公式/限值和证据要求的明文标准库。这种卡片化结构比普通 RAG 文档片段更强调依据身份和审核用途，也比完整知识图谱更易在小样本研究中落地验证。

### 2.3 类案推理、经验学习与智能体长期记忆研究

环评审核并非只依赖明文标准。实践中，经办人常根据同类项目经验识别报告中的隐性问题，例如胶水 VOCs 含量与指南限值不一致、活性炭更换周期虚长、非全密闭工序套用过高收集效率、总量替代方案证据不足、复合工艺产污环节遗漏等。这类经验通常存在于修改意见、批注、终稿变化和审批实践中，具有强烈的场景性和条件性。

类案推理（Case-Based Reasoning, CBR）为这种经验利用提供了经典理论基础。Watson 和 Marir（1994）指出，CBR 通过检索、复用、修正和保留既有案例来解决新问题，尤其适合难以完全显式建模、但存在丰富历史案例的领域。与基于规则的专家系统相比，CBR 可以减少完整知识规则抽取的负担，通过案例特征和相似性判断实现经验迁移。Thai 等（2022）进一步在不完整知识库问答场景中引入 case-based reasoning，说明当知识库稀疏或持续变化时，基于案例链路的非参数推理可以补充显式知识的不足。

近年的 LLM 智能体研究进一步将“经验”扩展为可外化、可存储和可检索的自然语言记忆。Zhao 等（2023）提出 ExpeL，使智能体从任务经历中抽取自然语言 insights 并在推理时调用，以避免每次任务都从零开始。Allard 等（2026）的 Experiential Reflective Learning 强调，经验并非越多越好，关键在于将过往轨迹反思为可迁移 heuristics，并在新任务中选择性检索相关经验。Du（2026）对 LLM agent memory 的综述也指出，长期记忆需要经过 write--manage--read 过程，并处理写入过滤、矛盾管理、选择性召回、延迟成本和隐私治理等问题。

这些研究为本研究的“经验库”提供了方法支撑：历史审核意见可以被视为一种面向任务的外部经验记忆，但其使用必须经过证据分级和 relevance 过滤。与通用智能体经验学习不同，本研究的经验并非由模型自我生成，而是来自顺德类案中的真实修改意见、批注和报告修订链条；经验卡也不能被无条件注入提示词，而必须标注适用行业、工艺模块、任务类型、证据状态和禁止使用场景。由此，经验库的作用不是替代标准库作出判断，而是在标准依据之外提供“类似项目经办人可能追问什么”的补充线索。

### 2.4 人机协同、公共部门 AI 与智能审核评价

环评审核属于公共治理和行政技术审查场景，AI 系统的评价不能只看答案是否接近标准答案，还应考虑人机关系、证据责任和误用风险。Fragiadakis 等（2025）指出，人机协同评价不仅涉及系统性能，还涉及交互质量、信任、适应性、任务分工和协作目标等多维因素，并提出结合定量和定性指标的评价框架。Alon-Barkat 和 Busuioc（2023）从公共部门决策角度研究算法建议与人类判断之间的关系，讨论 automation bias 和 selective adherence 等问题，提醒公共部门 AI 不能简单假定为中立和无风险的决策工具。

这些研究对环评智能审核具有直接启示。首先，环评审核中的 AI 应定位为辅助工具，其价值在于帮助经办人更快发现证据缺口、标准适用问题和潜在遗漏，而不是替代经办人的专业判断和行政责任。其次，评价指标应同时关注判断正确性、依据引用、证据使用、意见可操作性和经验使用边界。本研究 v0.6a 实验据此将评分拆分为 common_score_6 和 knowledge_use_score_4：前者衡量判断、证据和可操作性，后者衡量标准库和经验库的知识支撑。这样的设计与人机协同评价中的多维指标思想一致，也有助于避免将模型输出简单归结为“对/错”。

同时，公共部门 AI 风险研究也提醒本研究必须保持结论克制。当前 v0.6a benchmark 只有 18 道正式题和 2 道备选题，样本集中于佛山市塑胶行业指南适用场景；评分尚未完成独立双评和一致性检验；经验卡中仍有 pending 和 candidate 状态。因此，实验结果只能支持“方法链路可运行”“标准库提升依据可追溯性”“经验库在部分题目中带来经办人式追问增量”“relevance 机制有助于降低经验误用”等有限结论，不能表述为系统已被统计证明有效，更不能声称其可以替代人工审核。

### 2.5 研究述评与本文定位

综上，现有研究已经从不同侧面为环评智能审核提供了基础。EIA-LLM 研究证明了大模型在筛查、许可问答和公众审查中的应用潜力；EIA ontology、知识图谱和监管 NLP 研究说明了结构化知识对专业文本任务的重要性；CBR、经验学习和智能体记忆研究证明了历史经验可被外化为可复用的推理资源；人机协同与公共部门 AI 研究则为智能审核系统的评价和边界设定提供了规范性约束。

然而，既有研究之间仍存在明显断裂：一方面，环评 AI 研究多侧重文档问答、筛查或公众审查，尚未充分进入报告表技术审核中“标准依据--报告证据--类案经验”交织的细粒度场景；另一方面，知识图谱和智能体记忆研究虽然提供了结构化知识和经验复用的通用方法，但较少解决行政审核中经验证据等级、适用边界和误用控制问题。因此，本研究提出面向环评智能审核的“标准--经验双库”框架：以标准库回答“依据是什么、标准如何规定、报告应提供哪些证据”，以经验库回答“类似项目中经办人通常会追问什么、哪些问题容易漏审”，并通过 relevance 机制控制经验调用范围。

在研究对象上，本文不讨论泛泛塑料行业或普通 C2929 纯注塑项目，而聚焦佛山市塑胶行业指南中更具适用性的涉胶水、涂胶、贴合/复合、熟化和 VOCs 治理项目。在验证方式上，本文通过 A/B/C 三组最小实验比较“仅报告”“报告+标准库”“报告+标准库+经验库”的输出差异，以初步检验标准库和经验库对审核意见可追溯性和经办人式追问能力的增量作用。由此，本文的贡献不在于证明一个通用自动审批系统已经成熟，而在于提出并验证一条可追溯、可分层、可过滤的环评智能审核知识组织路径。

## 可压缩版研究空白表述

现有环评智能化研究已证明 LLM 可用于 EIA 筛查、许可问答和公众审查，但多数研究仍停留在文档理解或信息检索层面，尚未充分处理报告表技术审核中标准依据和地方审核经验并存的问题。知识图谱、本体和监管 NLP 研究强调了结构化知识的重要性，但较少面向环评经办人的具体审核动作。类案推理和智能体记忆研究说明历史经验可以被抽取和复用，却没有解决行政审核场景下经验证据等级、适用范围和误用过滤问题。因此，本文以佛山市塑胶行业建设项目环评审核为例，构建标准库与经验库相区分的双库框架，并通过 relevance 过滤和 A/B/C 最小实验，探索标准依据与类案经验协同支持智能审核的可行路径。
