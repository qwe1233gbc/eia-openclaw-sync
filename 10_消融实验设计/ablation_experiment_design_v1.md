# 消融实验设计 v1

## 实验组别

| 组别 | 输入条件 | 说明 |
|------|---------|------|
| **A组** | LLM (裸模型) | 仅给LLM环评报告原文，不提供任何外部知识 |
| **B组** | LLM + 法规库 | 给LLM报告 + 法规库检索结果（标准条款、限值） |
| **C组** | LLM + 技能库 | 给LLM报告 + 审核技能库（15个独立skill） |
| **D组** | LLM + 法规库 + 技能库 | 全量知识输入 |

## 实验假设

- **H1**: B组 > A组（法规库提供合规底线，减少基础性错误）
- **H2**: C组 > A组（技能库提供结构化审核流程，提升审核完整性）
- **H3**: D组 > B组 且 D组 > C组（法规+技能互补最优）
- **H4**: C组在"意见可执行性"维度优于B组（技能库输出更贴近经办人实际需求）

## 评分维度

| 维度 | 说明 |
|------|------|
| 问题识别 | 能否发现报告中的实质性问题 |
| 证据引用 | 审核依据是否可追溯到报告原文 |
| 逻辑完整性 | 问题→证据→标准→结论的链条完整度 |
| 意见可执行性 | 修改建议是否具体、可操作 |
| 误报控制 | 是否产生虚假或无依据的问题 |

## 测试集

- 当前: 126题 (seed_qa_126题_11份新增报告.xlsx)
- 目标: 扩展至~100份报告

## 执行方式

每组独立运行15个skill，汇总skill收集所有输出。
评分由人工或LLM-as-judge按五维度量规打分。
